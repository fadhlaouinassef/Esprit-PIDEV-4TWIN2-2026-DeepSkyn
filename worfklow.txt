DeepSkyn - Documentation du workflow n8n (quiz adaptatif)

1) Objectif du workflow
- Poser les questions du questionnaire de maniere progressive.
- Adapter la question suivante selon les reponses precedentes.
- Arreter le questionnaire quand il y a assez de signaux.
- Generer une analyse finale via Gemini.
- Retourner au frontend un objet standard: status, analysis, score, nextQuestion.

2) Point d'entree (n8n)
- Noeud: Quiz Webhook
- Methode: POST
- Path: skincare-quiz
- URL locale typique: http://localhost:5678/webhook/skincare-quiz
- Le webhook recoit les donnees envoyees par l'application Next.js.

3) Liaison avec l'application Next.js
- Frontend questionnaire: src/app/user/questionnaire/page.tsx
  - Envoie les reponses a l'API interne Next: /api/quiz/n8n
- Proxy backend Next: src/app/api/quiz/n8n/route.ts
  - Recoit le payload du frontend.
  - Appelle le webhook n8n (production webhook).
  - Retourne la reponse n8n au frontend.

Flux applicatif simplifie:
UI (questionnaire) -> /api/quiz/n8n (Next) -> n8n webhook -> reponse n8n -> UI

4) Payload attendu depuis l'app
Champs importants envoyes a n8n:
- userId: identifiant utilisateur
- quizId: identifiant questionnaire
- sessionId: identifiant de session questionnaire
- answersSoFar: liste des reponses accumulees
- lastQuestionId, lastAnswer: derniere reponse soumise
- imageParts (optionnel): images pour analyse multimodale

5) Etapes des noeuds dans workflow.json

Noeud 2 - Sauvegarder Reponse
- Appelle l'API Next: POST http://localhost:3000/api/quiz/save-answer
- Sauvegarde la derniere reponse si presente.

Noeud 3 - Recuperer Toutes les Questions
- Appelle l'API Next: GET http://localhost:3000/api/quiz/questions
- Charge toutes les questions disponibles.

Noeud 4 - Determiner Prochaine Etape
- Cree un ordre melange reproductible par session (seed sur sessionId).
- Identifie les questions deja repondues.
- Renvoie:
  - status=continue + nextQuestion (si on continue), ou
  - status=complete (si fin atteinte)
- Fournit aussi answersJSON et questionsJSON pour la suite.

Noeud 5 - IF - Quiz Complet
- Condition: status == complete
- Branche true: analyse finale Gemini
- Branche false: decision adaptative locale pour la prochaine question

Noeud 8 - Decision Adaptative Locale
- Classe les questions restantes selon la pertinence semantique.
- Mesure la couverture des signaux (type de peau, UV, habitudes, etc.).
- Peut decider un arret dynamique si les informations sont suffisantes.

Noeud 10 - Parser Prochaine Question
- Normalise la sortie de la branche adaptative.
- Retourne soit continue + nextQuestion, soit complete.

Noeud 11 - Reponse Continue
- Formate une reponse stable (status, analysis, score, nextQuestion).

Noeud 6 - Gemini Analyse Finale
- Appelle l'API Gemini generateContent.
- Prompt base sur reponses + contexte questionnaire (+ images si presentes).

Noeud 7 - Parser Analyse Finale
- Parse le JSON renvoye par Gemini.
- Retourne format final: status, analysis, score.

Noeud 9 - Merge Reponses
- Fusionne la branche continue et la branche finale.
- Envoie la reponse finale au webhook (responseMode=lastNode).

6) Format de reponse n8n vers l'application
Cas continue:
- status: continue
- nextQuestion: { id, text, type, options, quizId }

Cas complete:
- status: complete
- analysis: texte d'analyse
- score: nombre 0..100

7) Conditions necessaires pour que tout fonctionne
- Next.js lance sur http://localhost:3000
- n8n lance sur http://localhost:5678
- Workflow n8n actif
- Le path du webhook doit etre skincare-quiz
- Cle Gemini valide (ou variable GEMINI_API_KEY correctement configuree)

8) Probleme courant: n8n affiche "waiting for trigger event"
- Cette vue correspond souvent au mode test manuel de l'editeur.
- Les appels reels de l'app passent par le webhook de production.
- Verifier les executions dans l'onglet Executions de n8n.

9) Diagnostic rapide
- Si /api/quiz/n8n repond mais pas d'analyse detaillee:
  - verifier quota Gemini (429 RESOURCE_EXHAUSTED)
  - verifier le noeud IF et la branche complete
  - verifier la cle API et les restrictions du projet
- Si aucune question ne revient:
  - verifier /api/quiz/questions
  - verifier la base et les quiz/questions existants
